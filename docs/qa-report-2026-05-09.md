## 2. Critical Bugs

### BUG-1: Search Suggestions Always Return Empty

- **Severity:** CRITICAL
- **Affected:** `/api/search/suggest`, `/api/search`
- **Evidence:** `GET /api/search/suggest?q=23` → `{"stocks":[],"podcasts":[],"episodes":[],"tags":[]}`
- **Root cause:** `@router.on_event("startup")` in `search.py:92` does **not** fire on routers — only on the FastAPI `app` object. The in-memory `SuggestionIndex` is never built. The fallback `asyncio.create_task(build_search_index())` inside `/suggest` is a race condition that doesn't block the response.
- **Impact:** Search — a core user-facing feature — is completely non-functional.
- **File:** `backend/src/routers/search.py:92-95`
- **Fix:** Move the startup task to `main.py` using the app lifespan, or call the builder inside the `startup_event` in `main.py`.

### BUG-2: Industry Heatmap Renders Blank

- **Severity:** CRITICAL
- **Affected:** `/industry` page ("標普 500 地圖")
- **Evidence:** The heatmap area is a blank rectangle.
- **Root cause:** `getTreeMapData()` in `sectorData.ts:69-72` returns `[]` with a comment "Placeholder — can be expanded with actual data."
- **Impact:** Feature is visible in navigation and renders nothing. Misleading.
- **File:** `frontend/src/services/mocks/sectorData.ts:69-72`
- **Fix:** Populate with S&P 500 sector data or hide the tab until data is available.

### BUG-3: 18 out of 51 Unit Tests Failing (35%)

- **Severity:** CRITICAL
- **Affected:** `tests/unit/test_graph_service.py`, `test_news_service.py`, `test_stock_service.py`
- **Evidence:** `pytest tests/unit/ -v` → 18 failed, 33 passed
- **Root causes:**
  1. `GraphService.create_graph()` is async but tests call it synchronously → "coroutine was never awaited"
  2. `DataCollectionService` has no `mock_service` attribute → `test_get_sorted_stocks` returns 0 items
  3. `NewsService` tests use assertions that don't match current service implementation
- **Impact:** No safety net for regressions.
- **Files:** `tests/unit/test_graph_service.py`, `tests/unit/test_stock_service.py`, `tests/unit/test_news_service.py`

### BUG-4: Backend CI Never Blocks PRs

- **Severity:** CRITICAL
- **Affected:** `.github/workflows/backend-ci.yml`
- **Evidence:** `continue-on-error: true` on the test job + `pytest ... || echo "::warning::..."` swallows failures.
- **Impact:** Broken code merges freely. The CI pipeline is decorative.
- **File:** `.github/workflows/backend-ci.yml:10-11, 46`
- **Fix:** Remove `continue-on-error: true` and the `|| echo` fallback.

### BUG-5: Graph Gallery Zod Validation Errors

- **Severity:** CRITICAL
- **Affected:** `/story` page
- **Evidence:** Console error: `[API] Schema validation failed: data.nodes.0.data.marketCapTier: Invalid input`
- **Root cause:** `GraphNodeDataSchema` requires `marketCapTier: z.enum(['large', 'medium', 'small'])` but the API returns values outside this enum (or missing). `ticker` field is also missing on some nodes. A fallback masks this.
- **File:** `frontend/src/validation/schemas.ts:46-49`
- **Fix:** Make `marketCapTier` and `ticker` optional, or update API to always return valid values.

## 3. Medium Bugs

### BUG-6: Stock Page Title Is Doubled

- **Severity:** MEDIUM
- **Affected:** `/stock/:ticker` pages
- **Evidence:** Browser tab shows `"2330 (2330) - ... | TinBoker | TinBoker"`
- **Root cause:** SEO component appends "| TinBoker" and either a base `<title>` tag or Helmet defaults add it again. The `(2330)` duplication likely comes from structured data injection.
- **File:** `frontend/src/pages/StockDashboard.tsx:436`, `frontend/src/components/common/SEO.tsx`

### BUG-7: Fabricated Stock Key Statistics

- **Severity:** MEDIUM
- **Affected:** Stock dashboard sidebar ("關鍵數據")
- **Evidence:** Open = price × 0.98, High = price × 1.02, Low = price × 0.96, P/E hardcoded to 15.4
- **Impact:** Users see fake financial data. For a financial platform, this is misleading.
- **File:** `frontend/src/pages/StockDashboard.tsx:211-219`
- **Fix:** Use actual OHLC data from `stockData.chartData` or the API response.

### BUG-8: Debug Logs Leaking to Production Console

- **Severity:** MEDIUM
- **Affected:** All pages in production
- **Evidence:** `console.warn("[DEBUG] Environment check: ...")`, `console.warn("[StockHeaderCard] Fetching stock data for: ...")`
- **Impact:** Clutters user console, leaks implementation details.
- **Fix:** Gate behind `import.meta.env.DEV`.

### BUG-9: Stale CORS Origins

- **Severity:** MEDIUM
- **Affected:** `backend/src/config.py:104-113`
- **Evidence:** CORS origins include `trendbrief.xyz` (old domain) but not `tinboker.com` (current domain).
- **Note:** The regex pattern catches `*.vercel.app` and `*.pages.dev` but not `tinboker.com`. CORS may only work because Cloudflare proxies mask the origin.
- **Fix:** Replace `trendbrief.xyz` entries with `tinboker.com`, `dev.tinboker.com`, `staging.tinboker.com`.

### BUG-10: Recommendation Endpoint 404

- **Severity:** MEDIUM
- **Affected:** Stock dashboard "分析師觀點" section
- **Evidence:** Frontend calls `/api/recommendations/ticker/2330` but the actual route is `/api/recommendations/by-ticker/2330`.
- **Impact:** Analyst insights section always shows "此標的暫無詳細分析觀點" even when data exists.
- **File:** `frontend/src/services/recommendationService.ts`

### BUG-11: Health Endpoint Leaks Redis Connection String

- **Severity:** MEDIUM
- **Affected:** `/health` endpoint
- **Evidence:** Response includes `"connection_string": "redis://redis:6379/0"` exposing internal infrastructure.
- **Fix:** Remove or always mask the connection string in non-development environments.

## 4. Low-Priority Issues

### BUG-12: Deprecated `@app.on_event("startup")`

- **Severity:** LOW
- **Affected:** `backend/src/main.py:49`
- **Details:** FastAPI deprecated `on_event` in favor of `lifespan` context manager.
- **Fix:** Migrate to `@asynccontextmanager` lifespan pattern.

### BUG-13: Blocking `time.sleep(2)` in Auth

- **Severity:** LOW
- **Affected:** `backend/src/utils/auth.py:77`
- **Details:** Synchronous `time.sleep(2)` blocks the async event loop during clock-skew retry for Google token verification. Under load, this freezes all concurrent requests for 2 seconds.
- **Fix:** Use `await asyncio.sleep(2)` or remove the retry logic.

### BUG-14: Duplicate `pytest` in requirements.txt

- **Severity:** LOW
- **Affected:** `backend/requirements.txt:3,17`
- **Details:** `pytest>=7.4.0` (line 3) and `pytest==7.4.3` (line 17) — conflicting specs.
- **Fix:** Keep only one entry.

### BUG-15: API Client 150-Second Timeout

- **Severity:** LOW
- **Affected:** `frontend/src/services/api/client.ts:179`
- **Details:** Axios timeout is 150,000ms (2.5 minutes). Users will stare at loading spinners for 2.5 minutes before seeing an error.
- **Fix:** Reduce to 30s for most endpoints, with per-request overrides for known slow endpoints.

## 5. Infrastructure Issues

### INFRA-1: Deploy Uses `git reset --hard` + `git clean -fd`

- **Affected:** `.github/workflows/backend-deploy.yml:190-193`
- **Details:** Nukes all local changes including `gcp-service-account.json` (restored immediately after). If the restore step fails, the deploy is bricked.
- **Fix:** Use `git pull --ff-only` or a dedicated deployment artifact strategy.

### INFRA-2: Hardcoded VPS IP in Documentation

- **Affected:** `backend/AGENTS.md`
- **Details:** `152.53.136.182` is visible in plaintext while used as a secret in workflows.
- **Fix:** Remove IP from docs, reference the secret name only.

### INFRA-3: No Rollback Strategy

- **Affected:** All deploy workflows
- **Details:** If health check fails after deploy, the workflow exits with error but does not roll back to previous image/container.
- **Fix:** Save previous image tag and restore on failure.

### INFRA-4: Redis Cache Hit Rate at 35%

- **Affected:** Production Redis
- **Evidence:** `/health` shows 66 hits / 122 misses = 35.1% hit rate
- **Details:** Indicates cache warming is ineffective or TTLs are too short.
- **Fix:** Audit cache keys and TTLs; consider pre-warming on startup.
